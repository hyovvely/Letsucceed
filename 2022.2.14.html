Spring Framework 개발 순서 1. SQL Table 작성 ``` 어떤 데이터를 저장하고 어떤 방식으로 관리할지 목록 작성 및 데이터 크기 관리 결정 ``` 2. DTO (VO) 작성 ``` 테이블 내에 들어갈 데이터들이 들어갈 개체 생성 및 프론트 단에서 이용할 데이터들 목록 생성 ``` 3. DAO, Service, SQL 구문 작성 ``` 데이터 검색을 위한 SQL 구문을 작성 SQL을 만들고 나서 그 과정을 DAO와 변수 이름을 맞추어 주기 ``` 4. Controller 작성 ``` 어떤 DTO를 넘겨주고 데이터 송신 방식을 get, post, delete 등으로 할지 결정 백엔드에서는 프론트 단에서 화면을 만들기 전까지 제대로 SQL 구문이 가는지 Postman을 이용해서 화면단으로 넘어가는 데이터 전송을 테스트 ``` 5. jsp css 화면단 연결 작성 ``` Controller에서 보내주는 DTO들을 사용해서 페이지 작성 프론트의 경우 데이터 없이 화면단을 완성 프론트에 데이터가 넘어오지 않는 경우 대부분 Controller의 문제 즉, Back의 잘못 ``` VO 는 데이터 저장하는 역할 table에 있는 필드 선언 자료형에 맞춰 게터/세터를 VO 데이터 저장 연동해서 써서 만들었던 데이터베이스 필드명을 그대로 맞추고 게터/세터로 설정 밖에서 받아온 값으로 객체에 해당하는 넘버값을 this. 으로. DAO는 VO를 사용해서 데이터를 저장하고 설정 DAO는 로직을! 로직은 클라이언트가 요청을 하고 프론트 컨트롤러가 사용자가 사용하기 적합한 컨트롤러를 찾아준다 즉, 어떤 컨트롤러를 찾아갈지 정해줌! Controller는 Service에 의존하고 , 서비스는 DAO에 의존한다, (서비스에서 DAO에게 뭘 해주라고 의존) DI 설정은 의존성 주입 !
